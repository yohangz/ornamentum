Translations

Translations can be uses to override data table visible properties in two special cases. those translatable properties are required to binding into translations property in table global level configuration (inside ng-data-table scope), in any developer preferred language. the properties that can be configure in translations are:

  • Translations for No Data
    • No Data Message Header
    • No Data Message Body
  • Translations for Pagination
    • Pagination Limit
    • Pagination Range

Basic Usage

As mention in the above translations can be override in data table global level configuration. by default translations are initialize to it's default pre-set values. 

{{ demoSnippet.translations }}

Simply bind DataTableTranslations type of object into translations property is enough to take the the control over translations.

As mention in the above there are two scenarios that translations are useful. Let's see how we can configure each demonstrated properties as we want.

Translations for No Data

The default template will be shown when data is not available by mentioning a header and the message body. In no data scenario, users can customize the message needs to be appear using translations property.

No Data Message Header

noDataMessageHeader: DataTableTranslations  
Default Value: 'Whoops!'

This configuration is used to set the message header to be shown in the table body, when there are no data to display. The default value for this property is 'Whoops!'

No Data Message Body

noDataMessageBody: DataTableTranslations  
Default Value: 'No data to display. Added data will appear here.'

This configuration is used to set the message body on the message to be shown in the table body, when there are no data to display. The default value for this property is 'No data to display. Added data will appear here.

Translations for Pagination

Pagination translations property can be used to customize the pagination naming, which allow user to override the pagination limit and range namings.

Pagination Limit

paginationLimit: DataTableTranslations  
Default Value: 'Limit'

This is used to customize the pagination limit naming. The default value for this config is 'Limit'.

Pagination Range

paginationRange: DataTableTranslations  
Default Value: 'Results'

This is used to customize the pagination range naming. The default value for this config is 'Results'.
